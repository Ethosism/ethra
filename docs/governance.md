# Governance

Ethra needs governance because a civilizational language cannot grow by private taste alone. It needs rules for adding roots, accepting compounds, borrowing terms, preserving registers, and changing grammar.

The machine-readable governance model is in `spec/governance.yaml`.

## Roles

```text
proposer         suggests a root, word, compound, corpus item, or grammar change
lexicographer    checks derivation, sense boundaries, examples, and collisions
domain steward   checks accuracy and coverage inside a domain
culture steward  checks moral grammar, relational grammar, memory, beauty, and civilizational fit
maintainer       merges accepted changes and keeps tests/specs valid
```

These roles are editorial functions, not social ranks.

## Term Lifecycle

```text
candidate    proposed but not accepted
provisional  accepted for experimental corpus use
accepted     stable lexicon entry
deprecated   replaced for new formal material
historical   preserved for older, dialectal, or poetic use
```

## Root Admission

A new root must cover a durable semantic field. It should not be invented for one disposable object if a compound will do.

Good root candidates:

- culturally central concepts
- high-frequency concepts
- concepts that need many derived forms
- concepts whose moral, civic, ritual, or intimate forms matter

Weak root candidates:

- one-off gadgets
- decorative synonyms
- direct English calques
- terms that duplicate an existing root without adding contrast

## Protected Architecture

Ethra must preserve:

- root-depth
- moral agency grammar
- relational pronouns
- register distinctions
- speech as binding
- the future as claimant
- memory as identity-bearing

No expansion should make those easier to ignore.
